# Static Application Security Testing (SAST)
-   Status: proposed
-   Deciders: Bastian Habib
-   Date: 2026-06-05

## Platform Name
CH Product Experience

## Demand Type
Strategic Demand

## Description
Need for a uniform Application Security Testing Tool for all development engagements at Bayer

## Objectives
Need for enforcement of Static Application Security testing for Bayer web application code.
This applies to all Software Development/ software applications engineered in Bayer.

CodeQL is the SAST tool feature in Github advanced security which has already been in limited use by Bayer for a few years now. In the past Bayer has had approximately 250 licenses available. There are 400 available today, and we have budgeted for additional licenses to cover any additional amount needed to ensure that web applications that are reachable via the public internet are also covered.

## Decision
Review

## Recommendations
Architecture council approved on the CodeQL tool to be part of all web development activities,
Auto-scans will be in-place from this week of July. Communications in this regard are already shared via the stakeholder kit presentations published in Bayer IT. Awareness campaigns are planned through Github, CSF open office hours. Initial emails will be sent to end-user communities and BEAT registered system owners.
The auto-scan reports will be available in centralized locations; the application owners should review the results and determine the current state of the code if it poses a security threat to Bayer's system and
